# SideChat

> Cursor 侧栏多通道 MCP 面板 — 在侧栏发消息，AI 在对话窗口响应，持续轮询不中断。

## 为什么需要 SideChat

Cursor 的 Agent 对话窗口每次只能处理一轮交互。SideChat 通过 MCP（Model Context Protocol）在侧栏和 Agent 之间建立持久通道，让你可以：

- **随时插话** — 不用等上一轮结束，侧栏发送即可
- **多通道并行** — 最多 10 路独立会话（MCP-1 到 MCP-10），各做各的事
- **多窗口隔离** — 不同工作区的 Cursor 窗口各自独立，互不干扰

## 功能一览

| 功能 | 说明 |
|------|------|
| 多会话通道 | 侧栏标签页切换，每路绑定一个 Cursor 对话 |
| 截图粘贴 | `Ctrl+V` / `Cmd+V` 直接粘贴截图 |
| 语音输入 | 点击麦克风按钮语音转文字（Windows 支持系统语音） |
| 简洁界面 | 清爽亮色主题，与 Cursor 界面融合 |
| 会话重命名 | 双击标签页自定义会话名称 |
| 草稿保存 | 输入框内容自动保存，重启不丢失 |
| 会话备忘 | 为每路会话添加备忘说明 |

## 快速上手

1. 安装插件，点击侧栏 SideChat 图标
2. 点 **「使用当前」** 或填写工作区路径，然后点 **「开始配置」**
3. 在侧栏输入消息，`Cmd+Enter` 发送
4. 在 Cursor 对话窗口中输入：`请使用 sidechat-1 的 check_messages`
5. AI 开始响应后，即可通过侧栏持续对话

SideChat 现为免费可用版本，无需激活密钥。

## 常见问题

- **MCP 连不上** — 确认已在 Cursor 对话中调用了 `check_messages`
- **消息没反应** — 检查通道号是否与侧栏标签一致
- **提示 taking longer** — 请确保网络稳定，SideChat 通过 MCP 持续轮询，网络不稳定容易导致连接中断
- **增删会话后不生效** — 需要重新点一次 **「开始配置」**
