# SideChat 打包发布流程

## 快速打包（3 步）

在项目根目录执行：

```bash
# 1. 安装依赖（首次或依赖变更后执行）
npm install

# 2. 编译 src/ → dist/（每次改动 src/ 里的代码后必须执行）
npm run compile

# 3. 打包 VSIX（加日期后缀方便区分版本）
npm run package && mv sidechat-*.vsix "sidechat-$(node -p "require('./package.json').version")-$(date +%Y%m%d).vsix"
```

## 版本号更新（每次打包必做）

打包前**必须**在 `package.json` 里递增 `version` 字段，例如 `1.6.8` → `1.6.9`。
Cursor 根据版本号判断是否需要更新扩展，版本号不变则不会识别为新版。

## 安装测试

```bash
# 在 Cursor 中安装 VSIX
# 方式一：命令面板 → Extensions: Install from VSIX...
# 方式二：
code --install-extension sidechat-x.x.x-xxxxxxxx.vsix
```

## 注意事项

- `src/` 是源码目录，`dist/` 是编译产物，扩展运行时加载的是 `dist/`
- 改完 `src/` 后**必须** `npm run compile` 才能生效
- `mcp-server/` 是独立的 MCP 服务端，不经过 TypeScript 编译，直接打包进 VSIX
- 打包命令 `vsce package --no-dependencies` 不会把 `node_modules` 打进去，安装时由扩展自行 `npm install`
